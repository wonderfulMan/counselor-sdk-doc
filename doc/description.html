# 说明

# 背景

辅导猫移动端页面需要接入今日校园 APP、微信公众号、微信企业号、学习强国、钉钉等平台（暂未实现）,需要帮助开发人员屏蔽多个平台 jssdk 接口不一致的问题，特此开发了 counselor-sdk 来屏蔽差异化，做到输入、输出格式一致，只需要调用即可，对于多端尽可能做到了统一，提升开发人员开发效率和差异捕捉问题。

# 设计理念

面对众多的平台提供的 sdk 能力，counselor-sdk 设计之初制定了如下方向

  1. 统一输出和输入
  2. 内部采用依赖注入、自动挂载、平台检查
  3. js 文件动态注入

首先在使用每个 jssdk 接口时，我们规定了输入格式、输出格式，让开发人员快速开发应用尽可能的在业务中无需判断平台调用不同的 Api 所有接口对象均传入对象来便于以后的扩展，所有 Api 均返回 promise。

counselor-sdk 定位为一个 library，让它更加轻量、更加易用和调试是在业务中或不可缺的部分

import { previewFiles } from "counselor-sdk";

const { code, data, message } = await previewFiles({
  url: "http://xxxx.xxx.xx/index.docx",
  filename: "示例",
});

以上返回 code、data、message 会在快速上手中介绍

当调用以上 Api 后，counselor-sdk 会根据平台自动切换对应的实现类进行返回

export const previewFiles = (args) => {
  const featureFunction = transformApiBindProps(innerPreviewFiles, args);

  // transformApiBindProps内部判断了平台是否为企业微信或者今日校园等平台 如果不是返回false，静默失败
  if (featureFunction) {
    return featureFunction();
  }
};

以上代码中 transformApiBindProps 完成了当前平台的绑定，具体细节请至开发 Api 中介绍
